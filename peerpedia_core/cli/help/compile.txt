Compile an article from Markdown to a rendered format like PDF, SVG, PNG,
or HTML.  This is useful for sharing your paper outside PeerPedia — for
submitting to journals, posting on preprint servers, or printing.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Compile to PDF (default format)
  peerpedia compile abc12345
  # → Compiled to PDF
  #     /Users/you/.peerpedia/articles/abc12345/output.pdf

  # Compile to HTML (for sharing on the web)
  peerpedia compile abc12345 --format html
  # → Compiled to HTML
  #     /Users/you/.peerpedia/articles/abc12345/output.html

  # Compile to SVG (for figures and presentations)
  peerpedia compile abc12345 --format svg
  # → Compiled to SVG
  #     /Users/you/.peerpedia/articles/abc12345/output.svg

  # Compile to PNG (for quick previews or thumbnails)
  peerpedia compile abc12345 --format png

  # Reference by title keyword
  peerpedia compile "radioactivity"
  # → Compiled to PDF
  #     /Users/you/.peerpedia/articles/def78901/output.pdf

  # Compile with JSON output
  peerpedia compile abc12345 --format pdf --json
  # → {"success": true, "output_path": ".../output.pdf", "format": "pdf"}

WHICH FORMAT TO USE
  • PDF — Best for journal submission, printing, and formal publication.
    Preserves layout and fonts exactly.
  • HTML — Best for sharing on the web or reading in a browser.
    Supports interactive elements and responsive layout.
  • SVG — Best for figures, diagrams, and vector graphics.
    Scales to any size without losing quality.
  • PNG — Best for quick previews and thumbnails.
    Renders as a fixed-resolution image.

The output file is saved alongside your article in ~/.peerpedia/articles/.

FLAGS
  --format    Output format. Choices: pdf (default), html, svg, png.
              Default: pdf. Shown in examples above.
  --json      Print output as machine-readable JSON. Default: off.
              Shown in example above.

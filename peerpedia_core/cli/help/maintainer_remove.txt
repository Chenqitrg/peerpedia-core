Remove a co-author from your paper.  The removed user loses the ability
to edit, publish, or manage the article.  The paper itself is not affected
— only that person's control over it is taken away.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Remove a co-author by @name
  peerpedia maintainer remove abc12345 --target-user @pierre

  # Remove by user ID
  peerpedia maintainer remove abc12345 --target-user def67890

  # Remove by title keyword
  peerpedia maintainer remove "Radioactivity" --target-user @pierre

  # Remove a co-author who left the project
  peerpedia maintainer remove abc12345 --target-user @previous_collaborator
  # → The paper stays as-is.  The removed user can no longer edit or publish.

RULES
  • Only existing maintainers can remove other maintainers.
  • You cannot remove yourself if you are the last maintainer — every
    article must have at least one maintainer.
  • The removed user can still read the article if it is published
    (it becomes public).  They lose the ability to edit or manage it.

FLAGS
  --target-user @name   Who to remove.  Can be their @name, user ID,
                        or a unique prefix.  Required.
                        Shown in all examples above.

  --json                Output the result as JSON instead of plain text.
                        Useful for scripts and automation.

  --rich                Force human-readable colored output (default when
                        you type the command directly).

SEE ALSO
  maintainer add     Add a co-author
  maintainer list    See all co-authors

Delete your PeerPedia account.  This is a soft delete — your user
record is deactivated but preserved so that papers you authored and
reviews you wrote still show your name.  Your local session file is
removed, and you will need to register or log in again to continue.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       -> you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       -> you can type:  peerpedia article list
                       -> or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --json means "show the output as JSON"
                       Some flags take a value (--name "My Name")
                       Others work alone (--json, --rich).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.

EXAMPLES
  # Delete your account.  PeerPedia will ask for your password to confirm.
  peerpedia account delete

  # Delete your account using the PEERPEDIA_PASSWORD env var so you are
  # not prompted interactively.  Use this in scripts or automation.
  PEERPEDIA_PASSWORD='s3kr1t!' peerpedia account delete

  # Delete and get machine-readable confirmation.
  peerpedia account delete --json
  # -> {"status": "deleted", "user_id": "abc12345-..."}

  # What data remains after deletion
  peerpedia account delete
  # -> After confirming, your account is deactivated.  Your name still
  #   appears on papers and reviews you authored, but your profile no
  #   longer shows up in searches.  Verify by searching for yourself:
  peerpedia account search --name "Marie Curie"
  # -> "No users found matching 'Marie Curie'"

  # What happens if you try to delete while not logged in:
  # PeerPedia replies: "Not logged in. Run 'peerpedia account login' first."

WHAT HAPPENS
  • Your session file (~/.peerpedia/session.json) is removed.
  • Your user record is marked inactive, not erased from the database.
  • Papers you authored still show your name.
  • Reviews you wrote still show your name.
  • You can register again with the same name if needed — it will be a
    new account with no connection to the old one.
  • Deletion requires your password to confirm it is really you.

TIPS
  • Deletion is permanent for your session — you will need to log in or
    re-register to use PeerPedia again.
  • Your published articles and reviews remain in the network.  Other
    researchers can still read and cite them.
  • If you are unsure whether you want to delete, you can simply stop
    using PeerPedia instead.  There is no automatic expiration.
  • To delete from the command line without interactive prompts, set
    the PEERPEDIA_PASSWORD environment variable.

FLAGS
  --json   Print the result as JSON instead of human-friendly text.
           Default: off.  Shown in the "Delete with --json" example.
  --rich   Print the result as formatted Rich text (the default).

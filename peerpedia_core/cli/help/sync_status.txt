Check your connection to a peer server and see what needs to be synced.
Tells you if the server is reachable and how many pending changes you have.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Check default server (PEERPEDIA_SERVER env var)
  peerpedia sync status
  # → Server: https://peer.example.com (online)
  #   Pending: 2 ops
  #   ⚠ 2 changes not yet synced. Run sync push.

  # Check a specific server
  peerpedia sync status --server https://peer.example.com
  # → Server: https://peer.example.com (online)
  #   Pending: 0 ops

  # Server is unreachable
  peerpedia sync status
  # → Server: https://peer.example.com (offline)
  #   Pending: 3 ops
  #
  # Check your internet connection or the server URL. If the server is
  # temporarily down, your changes are saved locally and will be sent
  # when the server comes back.

  # Understanding "pending" output
  peerpedia sync status --json
  # → {"server": "https://peer.example.com", "online": true, "pending": 5}
  # Pending means changes you made offline (new articles, edits, follows)
  # that haven't been sent to the server yet.

WHICH SERVER DOES IT CHECK?
  PeerPedia uses the server URL stored in the PEERPEDIA_SERVER
  environment variable. If you haven't set one and don't provide
  --server, the command will remind you.

  To set your server permanently, add this to your shell profile:
    export PEERPEDIA_SERVER=https://peer.example.com

WHAT TO DO WHEN A SERVER IS UNREACHABLE
  • Check your internet connection.
  • Verify the server URL is correct (typos happen!).
  • The server might be temporarily offline — try again later.
  • Your local work is never lost. Changes queue up until you
    can reach a server again.

FLAGS
  --server    Which server to check. Overrides PEERPEDIA_SERVER.
              Default: PEERPEDIA_SERVER env var. Shown in example above.
  --json      Print output as machine-readable JSON instead of
              formatted text. Useful for scripts. Default: off.
              Shown in example above.

SEE ALSO
  sync push          Send your changes to the server
  sync pull          Fetch updates from the server
  sync discover      Find new articles on the network

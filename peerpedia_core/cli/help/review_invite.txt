Invite someone to review your paper.  The invited user receives a
notification and can accept or decline.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Invite a reviewer by @name
  peerpedia review invite abc12345 --user @pierre
  # → @pierre receives a notification asking them to review your paper.

  # Invite by user ID
  peerpedia review invite abc12345 --user def67890
  # → Same as above, but using the user's ID. Use this if you copied
  #   the ID from a notification or search result.

  # Invite by partial ID (first few characters)
  peerpedia review invite abc12345 --user def6
  # → PeerPedia finds the user whose ID starts with "def6".
  #   Works if the partial ID is unique enough.

  # What happens when someone is invited
  # After you run the command, the invited user sees a notification:
  #   @author has invited you to review "Radioactivity" (abc12345).
  #   Use "peerpedia review accept abc12345" to accept or
  #   "peerpedia review decline abc12345" to decline.

  # What happens if the invited person declines
  # You receive a notification: @pierre declined your review invitation
  # for "Radioactivity". You can invite someone else instead — there
  # is no limit on how many people you can invite.

  # What happens if the person already reviewed your paper
  peerpedia review invite abc12345 --user @pierre
  # → If @pierre already reviewed this paper, PeerPedia says:
  #   "@pierre has already reviewed this paper. You cannot invite
  #   someone who has already submitted a review."

REQUIREMENTS
  • Your paper must be in the sedimentation pool (published for review)
    before you can invite reviewers.  Use "article publish" first.
  • You can only invite registered PeerPedia users.  Use "account
    search" to find users by name or field of study.

FLAGS
  Each flag is shown in the examples above.  This reference explains it
  in full — come back after you have tried the examples.

  --user <user>        Who to invite.  Can be an @name (like @pierre),
                       a full user ID, or a partial ID.
                       Default: (required, no default).  Shown in all
                       examples above.

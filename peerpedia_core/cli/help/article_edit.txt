Edit a paper you're working on.  Every edit creates a new saved version
— you can always go back to an earlier version if needed.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Open your editor to revise the paper
  peerpedia article edit abc12345
  # → Your editor opens with the current content.  Make changes, save, close.

  # Update content without opening the editor
  peerpedia article edit abc12345 \
      --content "# Revised Title\n\nUpdated introduction..." \
      --no-editor

  # Change only the title (keep existing content)
  peerpedia article edit abc12345 --title "A Better Title" --no-editor

  # Edit by title keyword
  peerpedia article edit "Radioactivity"

  # Edit title and body together (without the editor)
  peerpedia article edit abc12345 --title "A New Title" \
      --content "# Completely Revised\n\nNew content here..." \
      --no-editor
  # → Changes both the title and the body in one command.

  # Pre-fill the editor with starting content
  peerpedia article edit abc12345 --content "# Start with this draft"
  # → Your editor opens with the given content already filled in.
  #   The old content is replaced, and you can make more changes
  #   before saving.

WHAT HAPPENS
  1. A new version (git commit) is created with your changes.
  2. The commit is cryptographically signed with your key.
  3. The old version is preserved — use "article diff" to compare.

Only the paper's maintainers (authors and co-authors) can edit it.
Published papers that are edited re-enter a 3-day sedimentation period.

FLAGS
  Each flag is shown in an example above.  This is the full reference.

  --content "text"    Replace the paper body with this text.
                      Omit to open your editor with the current content.
                      Example: shown in "Update content without opening
                      the editor" above.

  --title "text"      Change the paper's title.  Use this to rename
                      your paper without changing the body.
                      Example: shown in "Change only the title" above.

  --no-editor         Skip opening the editor.  Use this when you are
                      providing --content inline, only changing the
                      title, or scripting.
                      Example: shown in "Update content without opening
                      the editor" above.

List all co-authors (maintainers) of a paper, showing their names,
consent status for publishing, and roles.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # See who maintains a paper
  peerpedia maintainer list abc12345
  # → Maintainers for "Radioactivity":
  #   @marie_curie   (owner, consented)
  #   @pierre_curie  (maintainer, consented)
  #   @paul          (maintainer, consent pending)

  # List by title keyword
  peerpedia maintainer list "Radioactivity"
  # → Same output — you can use the paper's title if you don't have the ID.

  # Understand what consent status means
  #   "consented"     → that person has approved publication
  #   "consent pending" → they haven't decided yet
  #   "owner"         → the person who created the paper (always a maintainer)
  # All must have "consented" before the paper can be published.

  # Check who still needs to consent (when preparing to publish)
  peerpedia maintainer list abc12345
  # → Look for "consent pending" — those are the people who need to act.
  #    Ask them to run:  peerpedia maintainer consent abc12345

  # List maintainers as JSON (for scripts or automation)
  peerpedia maintainer list abc12345 --json
  # → {"paper_id": "abc12345", "maintainers": [
  #      {"name": "@marie_curie", "role": "owner", "consented": true},
  #      {"name": "@pierre_curie", "role": "maintainer", "consented": true},
  #      {"name": "@paul", "role": "maintainer", "consented": false}]}

WHO COUNTS AS A MAINTAINER
  • The paper's creator (the "owner") is always a maintainer.
  • Any co-authors you add via "maintainer add" are maintainers.
  • All maintainers have equal rights — anyone can edit, publish,
    or add more co-authors.
  • There is no special "admin" role.  Everyone is equal.

FLAGS
  --json    Output the list as JSON instead of the table format.
            Useful for scripts and automation.

  --rich    Force human-readable colored output (default when
            you type the command directly).

SEE ALSO
  maintainer add     Add a co-author
  maintainer remove  Remove a co-author
  maintainer consent Give your consent to publish

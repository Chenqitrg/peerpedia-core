Push your local papers, reviews, and social graph changes to a peer server.
Use this after working offline — your changes are queued and sent when
you run sync push.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Push to the default server (PEERPEDIA_SERVER env var)
  peerpedia sync push
  # → Push 2 article(s): abc12345, def67890
  #   ✓ abc12345 → abcde001
  #   ✓ def67890 → defab002

  # Push to a specific server
  peerpedia sync push --server https://peer.example.com
  # → Push 1 article(s): abc12345

  # Nothing to push
  peerpedia sync push
  # → Nothing to sync.

  # Push after editing a paper offline
  # 1. Edit your paper while away from internet.
  # 2. Come online.
  # 3. Run sync push — it sends only the changes since
  #    your last sync, not the whole file.

  # Push with JSON output (for scripts)
  peerpedia sync push --json
  # → {"synced": true, "articles": ["abc12345"]}

WHAT HAPPENS
  1. PeerPedia checks which articles have local changes not yet on the server.
  2. It finds a common ancestor commit using k-exponential probe.
  3. It sends an incremental git bundle with only the new changes.
  4. The server records a witnessed_at timestamp for priority disputes.

If there is a conflict (both you and someone else edited the same paper),
sync push will report it and ask you to resolve it.

TIPS
  • Run sync status first to see if you have pending changes.
  • Push is safe — your local copies are never deleted or overwritten.
  • You only push YOUR changes, not other people's articles.

FLAGS
  --server    Which server to push to. Overrides PEERPEDIA_SERVER.
              Default: PEERPEDIA_SERVER env var. Shown in example above.
  --json      Print output as machine-readable JSON instead of
              formatted text. Useful for scripts. Default: off.
              Shown in example above.

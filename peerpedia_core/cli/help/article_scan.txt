Check all papers in the sedimentation pool and promote any that have
been there for 3+ days to "published" status.  This runs automatically
when you start PeerPedia, but you can also run it manually.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Scan and promote papers that are ready
  peerpedia article scan
  # → Output:
  #   3 papers promoted to published.
  #     2 still in review (need 1 more day).
  #     5 papers not yet ready for publication.

  # Scan when there is nothing to do
  peerpedia article scan
  # → Output:
  #   No papers ready for promotion.
  #   8 papers still in review.  None have reached 3 days yet.
  #   Nothing changed — all papers continue sedimenting.

  # Scan with JSON output (for scripts)
  peerpedia article scan --json
  # → {"promoted": 3, "pending": 2, "total_in_review": 10}
  #   Useful for automated workflows or checking from a script.

  # Scan before a presentation to make papers public now
  peerpedia article scan
  # → If any papers have been in review for 3+ days, this promotes
  #   them immediately. Run this before sharing your work with an
  #   audience so your latest results are visible.

AUTOMATIC SCANNING
  • PeerPedia runs article scan automatically when you start the CLI.
  • The REPL also scans periodically while you're using it.
  • Manual scans are only needed if you disabled auto-scan or are
    waiting impatiently for a specific paper to publish.

WHAT HAPPENS
  • Every paper in the sedimentation pool is checked.
  • Papers that have been in review for ≥3 days become "published".
  • Published papers are permanently visible and citable.

FLAGS
  This command has no special flags.  Only the global flags apply:

  --json              Output the scan results as JSON.
  --rich              Output as human-readable text (this is the default).
                      Example: shown in "Scan with JSON output" above.

SEE ALSO

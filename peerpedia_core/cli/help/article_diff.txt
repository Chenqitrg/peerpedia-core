Compare two versions of a paper — like "track changes" for your
academic writing.  See exactly what was added, removed, or modified.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # See what changed in the most recent edit
  peerpedia article diff abc12345 ~1 HEAD

  # Compare any two specific versions
  peerpedia article diff abc12345 abc12345 def67890

  # Go back further (3 edits ago vs now)
  peerpedia article diff abc12345 ~3 HEAD

  # Diff by title keyword (no ID needed)
  peerpedia article diff "Radioactivity" ~1 HEAD
  # → PeerPedia finds the paper by its title and shows what changed
  #   in the most recent edit.

  # See the full history from first draft to current version
  peerpedia article diff abc12345 <first-commit-hash> HEAD
  # → Shows every change made since the paper was first created.
  #   Find the first commit hash using: article show --json

  # Diff output as JSON (for processing with other tools)
  peerpedia article diff abc12345 ~1 HEAD --json
  # → Returns the diff as structured JSON data instead of colored text.

UNDERSTANDING VERSION REFERENCES
  HEAD     = the latest (current) version
  ~1       = one version ago (the previous version)
  ~3       = three versions ago
  abc12345 = a specific version by its ID

  "~1 HEAD" means "compare the previous version against the current one"
  — this is the most common use: "what did I just change?"

The diff output highlights added lines in green and removed lines in red,
similar to what you'd see in a word processor's "track changes" mode.

FLAGS
  This command uses the global flags only:

  --json              Output the diff as structured JSON instead of
                      colored terminal text.  Good for programmatic use.
                      Example: shown in "Diff output as JSON" above.

  --rich              Output as human-readable text with colors
                      (this is the default).

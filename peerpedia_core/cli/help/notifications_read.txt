Mark a notification as read so it no longer appears in the default
(unread) view.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # Mark a notification as read (use the notification ID)
  peerpedia notifications read abcdef01
  # → Notification abcdef01 marked as read

  # First check your unread notifications to find the ID
  peerpedia notifications
  # → 2 unread notification(s)
  #
  #   abcdef01  2026-06-26 14:30  @pierre reviewed your paper  NEW
  #   abcdef02  2026-06-26 13:15  @marie replied to your review  NEW
  peerpedia notifications read abcdef01
  # → Notification abcdef01 marked as read

  # Mark a notification that doesn't exist
  peerpedia notifications read xyz99999
  # → Error: notification not found
  # Check the ID with 'peerpedia notifications'

  # Mark all current notifications as read (one by one)
  peerpedia notifications
  # → 3 unread
  peerpedia notifications read abcdef01
  peerpedia notifications read abcdef02
  peerpedia notifications read abcdef03
  peerpedia notifications
  # → No notifications.

HOW TO FIND THE NOTIFICATION ID
  The notification ID is the short code shown at the start of each
  line when you run 'peerpedia notifications'. It looks like
  "abcdef01" or "xyz12345".

  You can type just the first few characters of the ID if they
  are unique (e.g., "abc" instead of "abcdef01").

WHAT HAPPENS AFTER MARKING READ
  • The notification disappears from the default (unread) view.
  • It still appears when you run 'peerpedia notifications --all'.
  • Marking read only affects your local database — other users
    are not affected.
  • You cannot "unread" a notification (there is no undo).

FLAGS
  This command takes one required piece of information: the
  notification ID to mark as read. No additional flags needed.

See who follows a user on the network.  Shows the list of researchers
that are following a specific person.

HOW TO READ THIS HELP
  The examples below show real commands you can copy and paste.
  Everything you should type is shown — just replace the example values
  with your own.

    <angle brackets>   Replace with your own value.  Do NOT type the brackets.
                       peerpedia article show <id>
                       → you actually type:  peerpedia article show abc12345

    [square brackets]  This part is optional.  You can skip it entirely.
                       peerpedia article list [--mine]
                       → you can type:  peerpedia article list
                       → or:            peerpedia article list --mine

    --double-dashes    A "flag" (option) that changes what the command does.
                       --title means "the title comes next"
                       --scores means "the scores come next"
                       Some flags take a value (--title "My Paper")
                       Others work alone (--mine, --force, --feed).

  The prompt character > or $ in examples means "type this in your terminal."
  Don't type the prompt itself.


EXAMPLES
  # See your followers
  peerpedia followers --user @me
  # → Followers 5 user(s)

  # See someone else's followers (fetched from peer server)
  peerpedia followers --user @marie_curie --server https://peer.example.com
  # → Followers 142 user(s)

  # See followers using only local data (no network)
  peerpedia followers --user abc12345 --local
  # → Followers 3 user(s)

  # Followers with JSON output
  peerpedia followers --user @einstein --json
  # → [{"id": "...", "name": "Niels Bohr", ...}, ...]

WHEN TO CHECK FOLLOWERS
  • See how popular a researcher is (more followers = more influence).
  • Discover who's interested in a particular field.
  • Find potential collaborators — people following the same
    researcher may share your research interests.
  • Check if someone is following you after a sync.

LOCAL VS REMOTE
  By default, followers fetches data from the peer server (network).
  This gives you the most up-to-date information.

  Use --local to check your local database without making
  a network request. This is faster but may be outdated if
  you haven't synced recently.

FLAGS
  --user      Whose followers to view. Required (use @name, user ID,
              or UUID prefix). Shown in all examples.
  --server    Which peer server to fetch from. Overrides
              PEERPEDIA_SERVER env var. Default: PEERPEDIA_SERVER.
              Shown in example above.
  --local     Read from local database only — no network request.
              Default: off (fetches from server). Shown in example above.
  --json      Print output as machine-readable JSON. Default: off.
              Shown in example above.
